Creating a photo Matching Game

In this project, you'll learn how to create a simple memory matching game with images of famous persons, where the user clicks two cards to try to match them. If there's a match, the cards disappear; if not, the cards flip back. The code will be deployed and hosted in S3, and the deployment pipeline will be created using AWS CodePipeline, pulling code from GitHub and deploying it to S3 whenever a change is detected.

Step 1: Set Up Your Development Environment

  1. Create a GitHub Repository:
    1. Sign in to GitHub and create a new repository for your project.
    2. Initialize the repository with a README file and choose a .gitignore template for web development.
  2. Clone the Repository:
    1. Clone the repository to your local machine using git clone <repository-url>.

Step 2: Develop the Photo Matching Game

  1. Create the HTML Structure:
    1. Create an index.html file in your project directory.
    2. Add the basic structure of the HTML file, including a container for the game board.
  2. Style the Game:
    1. Create a styles.css file to style the game board and cards.
  3. Add the JavaScript Logic:
    1. Create a script.js file to handle the game logic.
  4. Add Images:
    1. Create an images folder in your project directory.
    2. Add images with names corresponding to those in the photos array in script.js.

Step 3: Deploy the Game to Amazon S3

  1. Create an S3 Bucket:
    1. Open the Amazon S3 console and click "Create Bucket."
    2. Name the bucket (e.g., "photo-matching-game") and keep the settings as default.
    3. Upload your project files (index.html, styles.css, script.js, and the images folder) to the bucket.
  2. Make the Bucket Public:
    1. Go to the bucket permissions and edit the bucket policy to allow public read access.
  3. Enable Static Website Hosting:
    1. In the bucket properties, enable static website hosting.
    2. Set the index document to index.html.

Step 4: Set Up AWS CodePipeline for Continuous Deployment

  1. Create a Pipeline:
    1. Open the AWS CodePipeline console and create a new pipeline.
    2. Name the pipeline and choose a new service role.
  2. Add Source Stage:
    1. Select GitHub as the source provider and connect your GitHub account.
    2. Choose the repository and branch to use as the source.
  3. Add Deploy Stage:
    1. Select Amazon S3 as the deployment provider.
    2. Choose the S3 bucket you created for the game.
    3. Set the S3 object key to be the file paths where your project files will be deployed (e.g., index.html, styles.css, script.js, and the images folder).
  4. Review and Create Pipeline:
    1. Review the pipeline settings and create the pipeline.
    2. The pipeline will automatically deploy your code to the S3 bucket whenever a change is detected in the GitHub repository.

Step 5: Test the Game

  1. Open the URL of your S3 bucket (the one provided in the static website hosting settings).
  2. Ensure that the game loads correctly and functions as expected.
  3. Make changes to your code and push them to GitHub to see the pipeline in action.
